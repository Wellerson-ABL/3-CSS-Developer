Layouts flexíveis - Multicolunas

		- O layout de múltiplas colunas é uma técnica de design responsivo que permite que o conteúdo
		   seja exibido em várias colunas, dependendo do tamanho da tela.
		
		- Isso pode ser feito usando CSS Grid ou Flexbox, e é uma maneira eficaz de organizar o conteúdo em telas
		  maiores,  enquanto ainda permite que ele seja exibido em uma única coluna em telas menores.

		- Exemplo de CSS Grid:
			.container {
				display: grid;
				grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
				gap: 10px;
			}

			
		- Exemplo de Flexbox:
			.container {
				display: flex;
				flex-wrap: wrap;
			}
			.item {
				flex: 1 1 200px;
				background-color: lightgray;
				padding: 20px;
			}

		- O layout de múltiplas colunas é uma técnica poderosa para criar layouts responsivos que se adaptam a 
		  diferentes tamanhos de tela e dispositivos.

		- No entanto, é importante lembrar que o layout de múltiplas colunas pode não ser adequado para todos 
	 	  os tipos de conteúdo, e deve ser usado com cuidado para garantir que o conteúdo seja legível e acessível
	          em todas as telas.

			viewport:
				- A viewport é a área visível de uma página da web em um dispositivo. Em dispositivos móveis, 
				 a viewport é geralmente menor do que em desktops, o que significa que o layout deve ser 
				 adaptado para caber na tela.

				- Para garantir que o layout seja responsivo, é importante usar unidades relativas, como 
				porcentagens e unidades de viewport (vw e vh), em vez de unidades absolutas, como pixels.
				
				- Isso permite que o layout se ajuste automaticamente ao tamanho da tela, garantindo que
				 o conteúdo seja exibido corretamente em todos os dispositivos.

    Meta tag viewport

    
		- A meta tag viewport é uma tag HTML que define a largura e a escala da viewport em dispositivos móveis.
		
		- Isso é importante para garantir que o layout seja responsivo e se ajuste corretamente ao tamanho da tela.

		- A meta tag viewport deve ser incluída no cabeçalho do documento HTML, e deve ter o seguinte formato:

			< meta name="viewport" content="width=device-width, initial-scale=1.0">

		- Isso garante que a largura da viewport seja igual à largura do dispositivo, e que a escala inicial seja 1.0.

		Quais:
			- width=device-width: sobrescreve a configuração padrão da viewport(navegadores), definindo a largura da viewport como a largura do dispositivo.
			- height=device-height: define a altura da viewport como a altura do dispositivo.

			- initial-scale=1.0: define o zoom inicial da pagina.

			- user-scalable=no: desabilita o zoom na pagina.
			- user-scalable=yes: permite o zoom na pagina.

			- maximum-scale=1.0: define a escala máxima(zoom) da viewport(pagina) como 1.0.
			- minimum-scale=1.0: define a escala mínima(zoom) da viewport(pagina) como 1.0.



    O que são Media Queries

    		>>> ESTUDAR MAIS SOBRE MEDIA QUERIES <<<
		
		- Media queries são uma técnica de design responsivo que permite aplicar estilos CSS diferentes com base nas características
		 do dispositivo, como largura da tela, altura, resolução e orientação.

		- Isso é feito usando a regra @media no CSS, que permite definir estilos específicos para diferentes tamanhos de tela e 
		dispositivos.

		- As media queries são uma parte essencial do design responsivo, pois permitem que o layout se adapte a diferentes tamanhos
		 de tela e dispositivos.

		- Isso garante que o conteúdo seja exibido corretamente em todos os dispositivos, independentemente do tamanho da tela ou 
		da resolução.

		quais são:
			- @media screen: aplica estilos apenas em telas de dispositivos.
			- @media print: aplica estilos apenas em impressoras.
			- @media all: aplica estilos em todos os dispositivos.

			- @media (max-width: 600px): aplica estilos para telas com largura máxima de 600px.
			- @media (min-width: 600px): aplica estilos para telas com largura mínima de 600px.
			- @media (orientation: portrait): aplica estilos para telas em orientação retrato.
			- @media (orientation: landscape): aplica estilos para telas em orientação paisagem.

		onde usar?
			- As media queries podem ser usadas em qualquer lugar no CSS, mas geralmente são
			 colocadas no final do arquivo CSS para garantir que os estilos sejam aplicados corretamente.

			- As media queries também podem ser usadas em arquivos CSS externos,
				 o que permite que os estilos sejam aplicados a várias páginas da web.

				Como usar na CSS externo?
					
					@media (max-width: 600px) {
						body {
							background-color: lightblue;
						}
					}



    Estruturas

    
		@media(at-rule)

		O que é @media?
			- A @media é uma regra CSS que permite aplicar estilos diferentes com base nas características do dispositivo,
				 como largura da tela, altura, resolução e orientação.

			Exemplo com min e max:
				@media (min-width: 600px) and (max-width: 1200px) {
					/* Estilos para telas entre 600px e 1200px */
				}



    Media Types

    		- Os media types são categorias de dispositivos que podem ser usados para aplicar estilos diferentes com base no tipo de dispositivo.

		- Os media types mais comuns são:
			- all: aplica estilos em todos os dispositivos.
			- screen: aplica estilos apenas em telas de dispositivos.
			- print: aplica estilos apenas em impressoras.
			- speech: aplica estilos apenas em dispositivos de leitura de tela.

			- embossed: aplica estilos apenas em dispositivos de impressão em relevo.
			- projection: aplica estilos apenas em dispositivos de projeção.
			- handheld: aplica estilos apenas em dispositivos portáteis.
			- tv: aplica estilos apenas em dispositivos de TV.			
			- braille: aplica estilos apenas em dispositivos de leitura em braille.
			
			no Deve tools, você pode ver os media types disponíveis em um dispositivo específico.
				Navegador Chrome:
					- Abra o DevTools (F12 ou Ctrl + Shift + I)
					- Clique no ícone de três pontos no canto superior direito do DevTools
					- Selecione "More tools" e depois "Rendering"
					- Na seção "Media", você verá os media types disponíveis para o dispositivo.
					(print e screen são os mais comuns)



    Media Features (recursos de mídia)

    
		- As media features são características específicas do dispositivo que podem ser usadas para aplicar estilos diferentes com base nas características do dispositivo.

		- As media features mais comuns são:
			- width: largura da tela.
			- height: altura da tela.
			- min-width: largura mínima da tela.
			- max-width: largura máxima da tela.
			- min-height: altura mínima da tela.
			- max-height: altura máxima da tela.
			- orientation: orientação da tela (portrait ou landscape).
			- resolution: resolução da tela.
			- aspect-ratio: proporção da tela (largura/altura).
			- color: número de cores suportadas pelo dispositivo.
			- color-index: número de cores suportadas pelo dispositivo.
			- monochrome: número de cores em escala de cinza suportadas pelo dispositivo.

    Operadores

    		- Os operadores são usados para combinar media queries e aplicar estilos diferentes com base nas características do dispositivo.

		- Os operadores mais comuns são:
			- and: combina duas ou mais media queries.
			- or: combina duas ou mais media queries.
			- not: nega uma media query.



    Maneiras de usar as Media Queries

    		- As media queries podem ser usadas de várias maneiras, incluindo:
			- Inline: as media queries podem ser incluídas diretamente no HTML usando a tag < style>.
			- Interno: as media queries podem ser incluídas em um arquivo CSS interno usando a regra @media.
			- Externo: as media queries podem ser incluídas em um arquivo CSS externo usando a regra @media.

			- Exemplo de uso inline:
				< style>
					@media (max-width: 600px) {
						body {
							background-color: lightblue;
						}
					}
				< /style>



    Breakpoints

    		- Os breakpoints são pontos específicos em um layout onde o design muda para se adaptar a diferentes 
		  tamanhos de tela e dispositivos.



    Devemos sempre usar as media queries

    		- Não é necessário usar media queries em todos os casos, mas elas são uma ferramenta poderosa para criar layouts 
		  responsivos que se adaptam a diferentes tamanhos de tela e dispositivos.

		- No entanto, é importante lembrar que o uso excessivo de media queries pode tornar o código CSS mais complexo e 
		  difícil de manter.

		- Portanto, é importante usar as media queries com moderação e apenas quando necessário.


			Laura Kalbag:
				"Não devemos usar media queries para tudo, mas devemos usá-las quando necessário."
				um design acessível e responsivo deve ser o objetivo final.
				Devemos usar media queries para garantir que o conteúdo seja exibido corretamente em todos os dispositivos,
				 mas devemos evitar o uso excessivo de media queries para não tornar o código CSS mais complexo e difícil de manter.
